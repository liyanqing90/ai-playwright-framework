# 条件分支使用指南

## 概述

条件分支功能允许测试用例根据不同的条件执行不同的测试步骤，使测试流程更加智能和灵活。支持基于变量比较和UI元素状态的条件判断。

## 基本语法

```yaml
- if: "${{ condition_expression }}"
  then:
    - action: action_when_true
      # 条件为真时执行的步骤
  else:  # else部分是可选的
    - action: action_when_false
      # 条件为假时执行的步骤
```

## 条件表达式类型

### 1. 变量比较

支持各种变量比较操作：

```yaml
# 字符串比较
- if: "${{ ${user_role} == 'admin' }}"
  then:
    - action: navigate
      value: "/admin"

# 数值比较
- if: "${{ ${retry_count} < 3 }}"
  then:
    - action: retry_operation

# 布尔值判断
- if: "${{ ${is_premium_user} }}"
  then:
    - action: access_premium_features
```

### 2. UI元素状态判断

#### 2.1 元素存在性检查

```yaml
# 检查元素是否存在于DOM中
- if: "${{ element_exists('#success-message') }}"
  then:
    - action: log
      value: "操作成功"
  else:
    - action: log
      value: "未找到成功消息"

# 检查错误提示是否存在
- if: "${{ element_exists('.validation-error') }}"
  then:
    - action: handle_validation_error
```

#### 2.2 元素可见性检查

```yaml
# 检查元素是否可见
- if: "${{ element_visible('#loading-spinner') }}"
  then:
    - action: wait
      value: 3
    - action: log
      value: "等待加载完成"

# 检查模态框是否显示
- if: "${{ element_visible('.modal') }}"
  then:
    - action: click
      selector: ".modal .close-btn"
```

#### 2.3 元素启用状态检查

```yaml
# 检查按钮是否启用
- if: "${{ element_enabled('#submit-button') }}"
  then:
    - action: click
      selector: "#submit-button"
  else:
    - action: log
      value: "提交按钮未启用，检查表单填写"
```

#### 2.4 元素文本内容检查

```yaml
# 检查状态文本
- if: "${{ element_text('#status') == 'Ready' }}"
  then:
    - action: proceed_to_next_step

# 检查计数器值
- if: "${{ element_text('.counter') == '0' }}"
  then:
    - action: reset_counter
```

#### 2.5 元素属性值检查

```yaml
# 检查输入框是否禁用
- if: "${{ element_attribute('#input-field', 'disabled') == None }}"
  then:
    - action: fill
      selector: "#input-field"
      value: "test data"

# 检查链接地址
- if: "${{ element_attribute('a.download', 'href') != None }}"
  then:
    - action: click
      selector: "a.download"
```

#### 2.6 元素数量检查

```yaml
# 检查列表项数量
- if: "${{ element_count('.list-item') > 0 }}"
  then:
    - action: click
      selector: ".list-item:first-child"
  else:
    - action: log
      value: "列表为空"

# 检查是否有多个选项
- if: "${{ element_count('select option') > 1 }}"
  then:
    - action: select_option
      selector: "select"
      value: "option2"
```

### 3. 复合条件判断

支持使用逻辑运算符组合多个条件：

```yaml
# AND 逻辑
- if: "${{ element_visible('#modal') and element_text('#modal .title') == 'Confirm' }}"
  then:
    - action: click
      selector: "#modal .confirm-btn"

# OR 逻辑
- if: "${{ element_exists('.success') or element_exists('.warning') }}"
  then:
    - action: handle_result

# 复杂组合
- if: "${{ (${user_type} == 'admin' or ${user_type} == 'moderator') and element_visible('#admin-panel') }}"
  then:
    - action: access_admin_features
```

## 实际应用场景

### 场景1：智能表单填写

```yaml
test_cases:
  - name: "智能表单填写"
    steps:
      # 检查必填字段是否已填写
      - if: "${{ element_attribute('#email', 'value') == '' }}"
        then:
          - action: fill
            selector: "#email"
            value: "${user_email}"
      
      # 根据用户类型显示不同字段
      - if: "${{ ${user_type} == 'business' }}"
        then:
          - if: "${{ element_visible('#company-field') }}"
            then:
              - action: fill
                selector: "#company-field"
                value: "${company_name}"
      
      # 检查提交按钮状态
      - if: "${{ element_enabled('#submit') }}"
        then:
          - action: click
            selector: "#submit"
        else:
          - action: log
            value: "表单验证未通过，无法提交"
```

### 场景2：动态内容处理

```yaml
test_cases:
  - name: "动态内容处理"
    steps:
      - action: navigate
        value: "/dashboard"
      
      # 处理不同的页面状态
      - if: "${{ element_visible('.loading') }}"
        then:
          - action: wait
            value: 5
      
      # 检查是否有新消息
      - if: "${{ element_count('.notification') > 0 }}"
        then:
          - action: click
            selector: ".notification:first-child"
          - if: "${{ element_text('.notification-type') == 'error' }}"
            then:
              - action: handle_error_notification
            else:
              - action: handle_info_notification
      
      # 检查数据是否加载完成
      - if: "${{ element_count('.data-row') == 0 }}"
        then:
          - action: log
            value: "暂无数据"
        else:
          - action: verify_data_display
```

### 场景3：错误处理和重试

```yaml
test_cases:
  - name: "智能错误处理"
    steps:
      - action: set_variable
        variable_name: "retry_count"
        value: 0
      
      # 重试循环
      - action: for_each
        items: [1, 2, 3]
        steps:
          - if: "${{ element_exists('.error-message') }}"
            then:
              - action: log
                value: "发现错误，准备重试"
              - action: refresh_page
              - action: wait
                value: 2
            else:
              - action: log
                value: "操作成功，退出重试"
              - action: break  # 假设支持break操作
```

## 最佳实践

### 1. 选择器优化

```yaml
# 推荐：使用稳定的选择器
- if: "${{ element_visible('[data-testid=\"submit-button\"]') }}"

# 避免：使用易变的选择器
- if: "${{ element_visible('.btn.btn-primary.mt-3') }}"
```

### 2. 错误处理

```yaml
# 推荐：为UI检查添加备选方案
- if: "${{ element_exists('#primary-action') }}"
  then:
    - action: click
      selector: "#primary-action"
  else:
    - if: "${{ element_exists('#secondary-action') }}"
      then:
        - action: click
          selector: "#secondary-action"
      else:
        - action: log
          value: "未找到可用的操作按钮"
```

### 3. 性能考虑

```yaml
# 推荐：先检查快速条件
- if: "${{ ${skip_validation} }}"
  then:
    - action: skip_step
  else:
    - if: "${{ element_visible('.complex-component') }}"
      then:
        - action: complex_validation
```

### 4. 可读性优化

```yaml
# 推荐：使用描述性的变量名
- action: set_variable
  variable_name: "is_login_required"
  value: "${{ not element_exists('.user-profile') }}"

- if: "${{ ${is_login_required} }}"
  then:
    - action: perform_login
```

## 元素选择器解析

条件分支中的UI元素检查函数支持与测试步骤相同的选择器解析方式：

### 1. 直接CSS选择器
```yaml
- if: "${{ element_visible('#login-button') }}"
  then:
    - action: click
      selector: "#login-button"
```

### 2. 元素映射名称
```yaml
# 在 elements.yaml 中定义：
# AI营销助手总开关: //div[@class='main-section__title']//button[@role='switch']

- if: "${{ element_attribute('AI营销助手总开关', 'aria-checked') == 'true' }}"
  then:
    - action: click
      selector: AI营销助手总开关
```

### 3. 复杂选择器
```yaml
# 支持各种Playwright选择器语法
- if: "${{ element_exists('role=button[name=\"提交\"]') }}"
  then:
    - action: click
      selector: 'role=button[name="提交"]'
```

## 选择器一致性

条件分支中的选择器解析与测试步骤完全一致：

```yaml
test_data:
  test_example:
    steps:
      # 步骤中使用元素映射名称
      - action: click
        selector: 营销
      
      # 条件分支中使用相同的元素映射名称
      - if: "${{ element_attribute('AI营销助手总开关', 'aria-checked') == 'true' }}"
        then:
          - action: click
            selector: AI营销助手总开关
        else:
          - action: log
            value: "开关未开启"
```

这确保了：
1. **一致性**：条件判断和操作使用相同的元素定位方式
2. **可维护性**：元素定位器只需在一个地方维护
3. **可读性**：使用有意义的元素名称而不是复杂的选择器

## 调试技巧

### 1. 添加调试日志

```yaml
- action: log
  value: "检查条件: element_visible('#target') = ${{ element_visible('#target') }}"

- if: "${{ element_visible('#target') }}"
  then:
    - action: log
      value: "条件为真，执行then分支"
  else:
    - action: log
      value: "条件为假，执行else分支"
```

### 2. 分步验证

```yaml
# 将复杂条件分解为多个简单条件
- action: set_variable
  variable_name: "modal_visible"
  value: "${{ element_visible('#modal') }}"

- action: set_variable
  variable_name: "title_correct"
  value: "${{ element_text('#modal .title') == 'Confirm' }}"

- if: "${{ ${modal_visible} and ${title_correct} }}"
  then:
    - action: proceed_with_confirmation
```

## 注意事项

1. **性能影响**：UI元素检查会执行实际的DOM查询，频繁使用可能影响测试性能
2. **等待时机**：在检查元素状态前，确保页面已加载完成
3. **异常处理**：UI检查函数内置了异常处理，检查失败时返回默认值而不会中断测试
4. **选择器稳定性**：使用稳定的选择器，避免因页面样式变化导致条件判断失效

## 常见问题

**Q: 为什么element_visible返回false？**
A: 可能原因：元素不存在、元素被隐藏、元素在视口外、页面未加载完成

**Q: 如何处理动态加载的元素？**
A: 在条件判断前添加适当的等待，或使用重试机制

**Q: 条件表达式中可以使用哪些运算符？**
A: 支持比较运算符（==, !=, <, >, <=, >=）、逻辑运算符（and, or, not）、算术运算符（+, -, *, /, %）

**Q: 如何在条件中使用正则表达式？**
A: 目前不直接支持正则表达式，可以使用字符串包含检查或精确匹配
