## 条件分支执行增强

### UI元素状态判断

除了基本的变量比较，条件分支现在支持基于UI元素状态的智能判断：

#### 支持的UI检查函数

| 函数 | 用途 | 示例 |
|------|------|------|
| `element_exists(selector)` | 检查元素是否存在 | `element_exists('#submit-btn')` |
| `element_visible(selector)` | 检查元素是否可见 | `element_visible('.modal')` |
| `element_enabled(selector)` | 检查元素是否启用 | `element_enabled('#input')` |
| `element_text(selector)` | 获取元素文本 | `element_text('.status') == 'ready'` |
| `element_attribute(selector, attr)` | 获取元素属性 | `element_attribute('#link', 'href')` |
| `element_count(selector)` | 获取元素数量 | `element_count('.item') > 0` |

#### 智能测试流程示例

```yaml
test_cases:
  - name: "智能购物流程"
    steps:
      # 检查用户登录状态
      - if: "${{ element_exists('.user-menu') }}"
        then:
          - action: log
            value: "用户已登录"
        else:
          - action: navigate
            value: "/login"
          - action: perform_login

      # 检查购物车状态
      - if: "${{ element_count('.cart-item') > 0 }}"
        then:
          - action: navigate
            value: "/checkout"
        else:
          - action: log
            value: "购物车为空，添加商品"
          - action: add_products_to_cart

      # 根据支付方式显示不同选项
      - if: "${{ element_visible('#credit-card-option') }}"
        then:
          - action: select_payment_method
            value: "credit_card"
        else:
          - if: "${{ element_visible('#paypal-option') }}"
            then:
              - action: select_payment_method
                value: "paypal"
```

### 条件嵌套和复合判断

支持多层条件嵌套和复杂的逻辑组合：

```yaml
# 多层嵌套示例
- if: "${{ ${user_type} == 'premium' }}"
  then:
    - if: "${{ element_visible('#premium-features') }}"
      then:
        - if: "${{ element_count('.feature-item') >= 5 }}"
          then:
            - action: test_all_premium_features
          else:
            - action: test_available_features
      else:
        - action: log
          value: "高级功能区域未显示"
  else:
    - action: test_basic_features

# 复合条件示例
- if: "${{ (element_exists('.success-msg') or element_exists('.warning-msg')) and ${operation_completed} }}"
  then:
    - action: verify_operation_result
```
